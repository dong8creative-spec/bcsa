# 🔍 나라장터 결과 검증 가이드

## 📋 검증 방법

### 방법 1: 나라장터 공식 사이트에서 직접 확인

1. **나라장터 사이트 접속**: https://www.g2b.go.kr
2. **입찰공고 검색** 메뉴 클릭
3. **검색 조건 설정**:
   - 기관명: "부산" 또는 "부산광역시"
   - 날짜 범위: 최근 30일
4. **검색 실행**
5. **결과 확인**: 현재 시스템 결과와 비교

### 방법 2: 공고번호로 직접 확인

현재 시스템이 반환한 공고번호를 나라장터에서 직접 조회:

1. 나라장터 사이트 접속
2. 공고번호 검색: `R25BK01251002` (예시)
3. 해당 공고의 실제 기관명 확인
4. "부산"과의 연관성 확인

### 방법 3: API 직접 호출 비교

```bash
# 현재 시스템
curl "https://apibid-oytjv32jna-du.a.run.app/api/bid-search?insttNm=부산&inqryDiv=1"

# 나라장터 원본 API (공사)
curl "https://apis.data.go.kr/1230000/ad/BidPublicInfoService/getBidPblancListInfoCnstwkPPSSrch?ServiceKey=YOUR_KEY&insttNm=부산&inqryDiv=1&inqryBgnDt=202512300000&inqryEndDt=202601292359&type=json"
```

## 🔍 발견된 현상

### 현재 시스템 결과
- "부산" 검색 시 10개 결과 반환
- 하지만 실제 기관명은 "서울대학교", "경기도 평택시" 등 부산과 무관

### 나라장터 원본 API 결과
- "부산" 검색 시 0개 결과
- 나라장터 API는 정확한 기관명만 매칭 (부분 일치 불가)

## 💡 분석

### 가능한 원인

1. **나라장터 API 필터링 특성**
   - `insttNm` 파라미터는 정확 일치만 지원
   - "부산"이라는 단독 키워드로는 매칭 실패
   - 실제 기관명은 "부산광역시", "부산시청", "부산대학교" 등

2. **현재 시스템 동작**
   - 여러 API 엔드포인트(물품, 용역, 공사) 병합
   - 각 API에서 필터가 제대로 작동하지 않았을 가능성
   - 또는 캐시된 데이터 반환

3. **검증 필요사항**
   - 나라장터 공식 사이트에서 실제 "부산" 관련 기관명 확인
   - 해당 기관명으로 재검색하여 비교
