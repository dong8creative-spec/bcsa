# 로컬에서 입찰공고 기능 테스트하기

이 문서는 **본인 터미널**에서 개발 서버를 띄우고, 입찰공고·상세조건 등 기능을 직접 확인하는 방법입니다.

---

## 1. 터미널에서 개발 서버 실행

프로젝트 루트에서 아래를 실행하세요.

```bash
cd /Users/donghnc/Documents/bcsa
npm run dev
```

정상이면 예시처럼 나옵니다.

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://127.0.0.1:3000/
  ➜  press h + enter to show help
```

서버를 종료하려면 터미널에서 `Ctrl + C` 를 누르세요.

---

## 2. 브라우저에서 접속

1. 브라우저를 열고 주소창에 **http://127.0.0.1:3000** 입력 후 접속  
   (Vite가 자동으로 브라우저를 띄웠다면 그 창을 사용해도 됩니다.)
2. 메뉴에서 **「입찰공고」** 를 클릭해 입찰공고 페이지로 이동합니다.

---

## 3. 테스트 체크리스트

### 3.1 기본 화면 (상세조건 접힌 상태)

- [ ] **처음 들어왔을 때** 아래만 보이는지 확인  
  - 입찰공고번호, 공고명, 공고종류  
  - 공고/개찰일자 (시작일·종료일), 입찰마감제외  
  - 「상세조건 펼치기」 링크  
  - **업무구분·업무여부·기관명·참조번호·참가제한지역은 보이지 않음**
- [ ] 하단에 **초기화**, **검색**, **상세조건** 버튼이 보이는지 확인

### 3.2 상세조건 펼치기/접기

- [ ] **「상세조건 펼치기」** 링크를 클릭  
  → 업무구분, 업무여부, 기관명, 참조번호, 참가제한지역, 업종, 추정가격, 세부품명번호, 조달요청번호, 쇼핑몰공고, 국내/국제, 계약유형, 계약법구분, 계약방법, 낙찰방법이 나타나는지 확인
- [ ] **「상세조건 접기」** 또는 하단 **「상세조건」** 버튼을 다시 클릭  
  → 위 항목들이 다시 숨겨지는지 확인
- [ ] 하단 **「상세조건」** 버튼으로도 펼치기/접기가 되는지 확인 (화살표 방향이 바뀌는지 확인)

### 3.3 검색

- [ ] 공고명에 **「대전」** 입력, 날짜 범위 설정(예: 2025-12-30 ~ 2026-01-29), 공고종류 **실공고** 선택 후 **「검색」** 클릭  
  → 결과 목록이 나오는지(또는 “검색 결과가 없습니다” 메시지가 나오는지) 확인
- [ ] **「초기화」** 클릭  
  → 입찰공고번호, 공고명, 기관명 등 입력값이 비워지는지 확인

### 3.4 API 연동 (배포 API 사용 시)

- `.env.development` 에 `VITE_API_URL=https://apibid-oytjv32jna-du.a.run.app` 로 되어 있으면, 검색 시 **배포된 API**를 사용합니다.
- 검색 후 브라우저 **개발자 도구(F12) → Console** 에서  
  `DEBUG_G2B_PARAMS`, `[TenderSearchFilter]` 관련 로그가 출력되는지 확인할 수 있습니다.

---

## 4. 포트가 이미 사용 중일 때

다른 프로그램이 3000번 포트를 쓰고 있으면 다음처럼 바꿀 수 있습니다.

```bash
# 예: 5173번 포트로 실행
npx vite --port 5173
```

이후 브라우저에서는 **http://127.0.0.1:5173** 으로 접속하면 됩니다.
