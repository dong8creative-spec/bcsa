# 로컬에서 입찰공고·프로그램 기능 테스트하기

본인 터미널에서 개발 서버를 띄우고, 입찰공고·프로그램 신청 등 기능을 직접 확인하는 방법입니다.

---

## 1. 터미널에서 개발 서버 실행

프로젝트 루트에서 아래를 실행하세요.

```bash
cd /Users/donghnc/Documents/bcsa
npm run dev
```

정상이면 예시처럼 나옵니다.

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://127.0.0.1:3000/
  ➜  press h + enter to show help
```

서버를 종료하려면 터미널에서 **Ctrl + C** 를 누르세요.

---

## 2. 브라우저에서 접속

1. 브라우저를 열고 주소창에 **http://127.0.0.1:3000** 입력 후 접속  
   (Vite가 자동으로 브라우저를 띄웠다면 그 창을 사용해도 됩니다.)
2. 메뉴에서 **「입찰공고」** 또는 **「프로그램」** 등 원하는 메뉴로 이동합니다.

---

## 3. 테스트 체크리스트

### 3.1 입찰공고 — 기본 화면

- [ ] **입찰공고 페이지** 진입 시 아래가 보이는지 확인  
  - 입찰공고번호, 공고명, 공고종류  
  - 공고/개찰일자 (시작일·종료일), 입찰마감제외 체크박스  
  - **초기화**, **검색** 버튼
- [ ] 공고명에 키워드 입력, 날짜 범위 설정 후 **검색** → 결과 목록 또는 "검색 결과가 없습니다" 확인
- [ ] **초기화** 클릭 → 입력값이 비워지고 날짜가 기본(최근 30일)으로 복원되는지 확인

### 3.2 입찰공고 — API 연동 (배포 API 사용 시)

- `.env.development` 에 `VITE_API_URL=https://apibid-oytjv32jna-du.a.run.app` 로 되어 있으면, 검색 시 **배포된 API**를 사용합니다.
- 검색 후 브라우저 **개발자 도구(F12) → Console** 에서  
  `DEBUG_G2B_PARAMS`, `[TenderSearchFilter]` 관련 로그가 출력되는지 확인할 수 있습니다.

### 3.3 프로그램 — 신청 (무료 / 유료)

- [ ] **프로그램** 메뉴에서 프로그램 카드의 **신청** 클릭
- [ ] 모달에서 **신청사유**, **사전질문 1**, **사전질문 2** 입력
- [ ] **무료 프로그램**: 버튼이 **「신청하기」** 로 보이고, 클릭 시 신청 완료
- [ ] **유료 프로그램**(Admin에서 신청 비용 입력한 경우): **「신청 비용: N원」** 이 보이고, 버튼이 **「결제하기」** 로 보이는지 확인

### 3.4 마이페이지·즐겨찾기

- [ ] 로그인 후 **입찰공고** 결과에서 즐겨찾기(별) 토글 → **마이페이지 → 즐겨찾기** 탭에서 목록·삭제 동작 확인

---

## 4. 포트가 이미 사용 중일 때

다른 프로그램이 3000번 포트를 쓰고 있으면 다음처럼 바꿀 수 있습니다.

```bash
# 예: 5173번 포트로 실행
npx vite --port 5173
```

이후 브라우저에서는 **http://127.0.0.1:5173** 으로 접속하면 됩니다.
