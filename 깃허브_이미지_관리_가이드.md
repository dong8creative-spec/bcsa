# 깃허브 이미지 관리 가이드

## 📋 개요

부산청년사업가들 웹사이트의 이미지들을 깃허브에서 관리하는 방법을 안내합니다. 깃허브는 무료로 이미지를 호스팅할 수 있으며, GitHub Pages를 통해 CDN처럼 사용할 수 있습니다.

---

## 🗂️ 이미지 폴더 구조

프로젝트의 이미지 폴더 구조는 다음과 같습니다:

```
부청사 홈페이지 작업/
├── assets/
│   └── images/
│       ├── favicon.ico              # 파비콘 (메인)
│       ├── favicon-32x32.png        # 파비콘 (32x32)
│       ├── favicon-16x16.png        # 파비콘 (16x16)
│       ├── apple-touch-icon.png     # iOS 아이콘 (180x180)
│       ├── logo/                    # 로고 이미지들
│       ├── programs/                # 프로그램 이미지들
│       ├── members/                 # 회원 프로필 이미지들
│       ├── reviews/                 # 후기 이미지들
│       ├── seminars/                # 세미나 이미지들
│       ├── food/                    # 맛집 이미지들
│       └── backgrounds/             # 배경 이미지들
```

---

## 🚀 깃허브에 이미지 업로드하기

### 방법 1: 깃허브 웹 인터페이스 사용

1. **깃허브 저장소 접속**
   - https://github.com/your-username/your-repo 접속
   - `assets/images/` 폴더로 이동

2. **새 폴더 생성 (필요한 경우)**
   - "Add file" > "Create new file" 클릭
   - 파일명에 `programs/test.png` 입력 (폴더 자동 생성)
   - 취소하고 "Upload files" 클릭

3. **이미지 업로드**
   - "Add file" > "Upload files" 클릭
   - 이미지 파일을 드래그 앤 드롭 또는 "choose your files" 클릭
   - 커밋 메시지 작성 (예: "Add program images")
   - "Commit changes" 클릭

### 방법 2: Git 명령어 사용

```bash
# 저장소 클론 (처음 한 번만)
git clone https://github.com/your-username/your-repo.git
cd your-repo

# 이미지 파일을 해당 폴더에 복사
# 예: assets/images/programs/seminar1.jpg

# 변경사항 추가
git add assets/images/programs/seminar1.jpg

# 커밋
git commit -m "Add seminar program image"

# 푸시
git push origin main
```

## 🔗 이미지 URL 사용 방법

### GitHub Pages 사용 (권장)

1. **GitHub Pages 활성화**
   - 저장소 Settings > Pages
   - Source를 "main" 브랜치로 설정
   - 저장소가 `username.github.io/repo-name` 형태로 공개됨

2. **이미지 URL 형식**
   ```
   https://username.github.io/repo-name/assets/images/programs/seminar1.jpg
   ```

3. **index.html에서 사용**
   ```html
   <img src="https://username.github.io/repo-name/assets/images/programs/seminar1.jpg" alt="세미나 이미지">
   ```

### Raw 파일 직접 사용

GitHub Pages 없이도 Raw 파일로 직접 접근 가능:

```
https://raw.githubusercontent.com/username/repo-name/main/assets/images/programs/seminar1.jpg
```

**주의**: Raw 파일은 용량 제한이 있고, 대용량 파일은 권장하지 않습니다.

## 📸 이미지 타입별 관리 방법

### 1. 파비콘 (Favicon)

**위치**: `assets/images/`

**필요한 파일**:
- `favicon.ico` (16x16, 32x32 포함)
- `favicon-32x32.png`
- `favicon-16x16.png`
- `apple-touch-icon.png` (180x180)

**생성 방법**:
1. 온라인 도구 사용: https://www.favicon-generator.org/
2. 원본 이미지 업로드 (최소 260x260px 권장)
3. 생성된 파일들을 다운로드
4. `assets/images/` 폴더에 업로드

**HTML에서 사용** (이미 적용됨):
```html
<link rel="icon" type="image/x-icon" href="/assets/images/favicon.ico">
<link rel="icon" type="image/png" sizes="32x32" href="/assets/images/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/assets/images/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/assets/images/apple-touch-icon.png">
```

### 2. 프로그램 이미지

**위치**: `assets/images/programs/`

**파일명 규칙**:
- `program-{id}-main.jpg` (대표 이미지)
- `program-{id}-{순서}.jpg` (추가 이미지)

**예시**:
- `program-1-main.jpg`
- `program-1-1.jpg`
- `program-1-2.jpg`

### 3. 회원 프로필 이미지

**위치**: `assets/images/members/`

**파일명 규칙**:
- `member-{id}.jpg` 또는 `member-{이름}.jpg`

**예시**:
- `member-1.jpg`
- `member-kim-minsu.jpg`

### 4. 후기 이미지

**위치**: `assets/images/reviews/`

**파일명 규칙**:
- `review-{postId}-{순서}.jpg`

**예시**:
- `review-7-1.jpg`
- `review-7-2.jpg`

### 5. 맛집 이미지

**위치**: `assets/images/food/`

**파일명 규칙**:
- `food-{id}.jpg`

**예시**:
- `food-1.jpg`
- `food-2.jpg`

### 6. 배경 이미지

**위치**: `assets/images/backgrounds/`

**파일명 규칙**:
- `hero-bg.jpg` (히어로 섹션)
- `stats-bg.jpg` (통계 섹션)
- `cta-bg.jpg` (CTA 섹션)

## 🔄 이미지 업데이트 방법

### 기존 이미지 교체

1. **깃허브 웹에서**
   - 해당 이미지 파일 클릭
   - 우측 상단 "..." 메뉴 > "Delete file" 클릭
   - 새 이미지 업로드

2. **Git 명령어로**
   ```bash
   # 기존 파일 삭제
   git rm assets/images/programs/old-image.jpg
   
   # 새 파일 추가
   git add assets/images/programs/new-image.jpg
   
   # 커밋
   git commit -m "Update program image"
   git push origin main
   ```

## 📦 이미지 최적화

### 파일 크기 최적화

깃허브는 대용량 파일에 제한이 있으므로 이미지를 최적화하는 것이 중요합니다.

**권장 사항**:
- 각 이미지: **5MB 이하**
- 총 저장소 크기: **100MB 이하** (권장)

**최적화 도구**:
1. **TinyPNG**: https://tinypng.com
2. **Squoosh**: https://squoosh.app
3. **ImageOptim**: https://imageoptim.com (Mac)

### 해상도 권장사항

| 이미지 타입 | 권장 해상도 | 파일 크기 |
|------------|------------|----------|
| 파비콘 | 32x32px, 16x16px | 10KB 이하 |
| 프로그램 이미지 | 1200x800px | 500KB 이하 |
| 프로필 이미지 | 400x400px | 200KB 이하 |
| 후기 이미지 | 1200x800px | 500KB 이하 |
| 배경 이미지 | 1920x1080px | 1MB 이하 |

## 🔐 Private 저장소에서 이미지 사용

### GitHub Pages 사용 불가

Private 저장소는 GitHub Pages를 사용할 수 없습니다. 대안:

1. **Raw 파일 직접 사용** (제한적)
   ```
   https://raw.githubusercontent.com/username/repo-name/main/assets/images/...
   ```
   - 인증이 필요할 수 있음
   - 대용량 파일 제한

2. **ImgBB 사용** (현재 방식)
   - 현재 가이드 참조: `이미지_등록_가이드.md`

3. **다른 이미지 호스팅 서비스**
   - Cloudinary
   - AWS S3
   - Imgur

## 🛠️ 문제 해결

### 이미지가 표시되지 않는 경우

1. **URL 확인**
   - GitHub Pages URL이 올바른지 확인
   - 파일 경로가 정확한지 확인 (대소문자 구분)

2. **파일 존재 확인**
   - 깃허브 저장소에서 파일이 실제로 존재하는지 확인
   - 파일명과 확장자가 정확한지 확인

3. **캐시 문제**
   - 브라우저 캐시 삭제 (Ctrl+Shift+Delete)
   - 하드 리프레시 (Ctrl+F5)

4. **CORS 문제**
   - GitHub Pages는 CORS를 지원하므로 문제 없음
   - Raw 파일 사용 시 CORS 문제 발생 가능

### 대용량 파일 업로드 실패

1. **Git LFS 사용** (100MB 이상 파일)
   ```bash
   # Git LFS 설치 후
   git lfs install
   git lfs track "*.jpg"
   git lfs track "*.png"
   git add .gitattributes
   git commit -m "Add Git LFS tracking"
   ```

2. **이미지 압축**
   - 위의 최적화 도구 사용
   - 해상도 조정

- [ ] 올바른 폴더에 업로드했는가?
- [ ] GitHub Pages가 활성화되어 있는가?
- [ ] HTML에서 URL이 올바르게 설정되어 있는가?
- [ ] 이미지가 실제로 표시되는지 확인했는가?
