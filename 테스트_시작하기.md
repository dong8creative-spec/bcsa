## 🎯 주요 테스트 시나리오

### 1️⃣ 회원 기능 테스트

1. **[회원가입]** 클릭
2. 테스트 계정 생성:
   ```
   이메일: test@example.com
   비밀번호: test1234
   이름: 테스트 사용자
   ```
3. 가입 완료 후 **[로그인]**
4. **[마이페이지]** 에서 정보 확인
5. 정보 수정 테스트
6. **[로그아웃]**

### 2️⃣ 커뮤니티 기능

1. **[커뮤니티]** 클릭
2. 게시글 목록 확인
3. 로그인 후 **[글쓰기]** 클릭
4. 게시글 작성 및 등록
5. 작성한 게시글 확인
6. 댓글 작성 테스트

### 3️⃣ 맛집 정보

1. **[부산맛집]** 클릭
2. 맛집 목록 확인
3. 맛집 카드 클릭하여 상세 보기
4. 카카오맵 표시 확인
5. 지도에서 위치 확인

### 4️⃣ 관리자 기능

```
http://127.0.0.1:5173/admin
```

- 관리자 권한이 있는 계정으로 로그인
- 대시보드 확인
- 콘텐츠 관리, 사용자 관리 등 테스트

## 🐛 문제 해결

### 1. 포트가 이미 사용 중인 경우
```bash
# 다른 포트 사용
npx vite --port 8080
```

### 2. 페이지가 로드되지 않는 경우
- 터미널에서 에러 메시지 확인
- 브라우저 캐시 삭제 (Ctrl + Shift + R)
- 서버 재시작

### 3. Firebase 연결 오류
- 인터넷 연결 확인
- Firebase Console에서 프로젝트 상태 확인
- `src/firebase.js` 설정 확인

### 4. API 호출 실패
- `.env.development` 파일 확인
- Network 탭에서 요청 상세 확인

### 5. 이미지가 표시되지 않는 경우
- `public/assets/images/` 폴더 확인
- 이미지 경로 확인
- 브라우저 캐시 삭제

## 💡 추가 팁

### HMR (Hot Module Replacement)
코드를 수정하면 브라우저가 자동으로 업데이트됩니다.
- 저장하면 즉시 반영
- 페이지 상태 유지
- 빠른 개발 가능

### 네트워크 요청 모니터링
Network 탭에서 확인할 수 있는 정보:
- 요청 URL
- 요청 메서드 (GET/POST/etc)
- 상태 코드 (200/404/500/etc)
- 응답 시간
- 요청/응답 헤더
- 페이로드 데이터
