# 테스트 가이드

## 0. 프로그램 상세 페이지(레이아웃) 테스트

로그인 없이 프로그램 상세 레이아웃을 확인할 때:

1. 로컬에서 개발 서버 실행:
   ```bash
   npm run dev
   ```
2. 브라우저에서 **테스트 전용 URL** 접속:
   ```
   http://localhost:5173/program/apply/test
   ```
3. mock 데이터로 상세 페이지가 표시됩니다. 우측 상단에 "테스트 화면" 배지가 보이면 테스트 모드입니다. 신청/결제 버튼은 눌러도 실제 신청되지 않고 안내만 표시됩니다.

## 1. 마이페이지 반응형 및 기능 테스트

### 테스트 시나리오

#### 1.1 레이아웃 테스트
1. 다양한 화면 크기에서 확인:
   - 모바일: 375px, 414px
   - 태블릿: 768px, 1024px
   - 데스크톱: 1280px, 1920px
2. 예상 결과:
   - 모든 화면 크기에서 레이아웃 깨짐 없음
   - 여백과 타이포그래피가 일관성 있게 유지
   - 미니멀 클린 스타일 유지

#### 1.2 탭 전환 테스트
1. "신청한 모임" 탭 클릭
2. "내 게시글" 탭 클릭
3. "본인인증 정보" 탭 클릭
4. "회사 소개" 탭 클릭 (후원 회원만)
5. 예상 결과:
   - 탭 전환이 부드럽게 작동
   - 각 탭의 컨텐츠가 정상 표시
   - 활성 탭 스타일 적용

#### 1.3 프로필 편집 테스트
1. "개인정보 수정" 버튼 클릭
2. 각 필드 수정:
   - 이름, 회사명, 직책
   - 업종, 주소, 전화번호
   - 프로필 이미지 업로드
3. "저장" 버튼 클릭
4. "취소" 버튼 테스트
5. 예상 결과:
   - 모든 필드 수정 가능
   - 저장 시 변경사항 반영
   - 취소 시 원래 값으로 복원

#### 1.4 게시글 편집 테스트
1. "내 게시글" 탭에서 편집 버튼 클릭
2. 제목, 내용 수정
3. 이미지 추가/삭제 (카테고리별)
4. "수정" 버튼 클릭
5. 예상 결과:
   - 모달이 정상적으로 열림
   - 모든 필드 수정 가능
   - 이미지 업로드/삭제 정상 작동
   - 변경사항 저장 후 모달 닫힘

#### 1.5 본인인증 정보 테스트
1. "본인인증 정보" 탭 클릭
2. 인증된 정보 확인:
   - 이름, 전화번호
   - 생년월일, 성별
   - 인증 거래번호
3. 예상 결과:
   - 모든 정보가 명확하게 표시
   - 미니멀한 디자인 유지
   - 정보 구분이 명확

#### 1.6 회사 소개 테스트 (후원 회원)
1. "회사 소개" 탭 클릭
2. 대표 이미지 URL 입력
3. 회사 소개 텍스트 작성
4. 추가 사진 URL 입력 (최대 3장)
5. "저장하기" 버튼 클릭
6. 예상 결과:
   - 모든 필드 입력 가능
   - 이미지 미리보기 표시
   - 저장 후 데이터 유지

## 2. 통합 테스트

### 2.1 전체 플로우
1. 메인 페이지 → 로그인 → 마이페이지 → 프로필 수정
3. 커뮤니티 → 게시글 작성 → 마이페이지에서 확인 → 편집
4. 프로그램 신청 → 마이페이지에서 확인 → 취소

### 2.2 성능 테스트
1. Lighthouse 점수 확인:
   - Performance: 80+
   - Accessibility: 90+
   - Best Practices: 90+
   - SEO: 90+
2. 로딩 시간:
   - 초기 로딩: 3초 이내
   - API 응답: 2초 이내 (캐시 미스)
   - API 응답: 500ms 이내 (캐시 히트)
