## 개발 환경 설정

### 필수 요구사항

- **Node.js**: v18 이상
- **npm**: v9 이상 (또는 yarn, pnpm)

### 의존성 설치

프로젝트 루트 디렉토리에서 다음 명령어를 실행하세요:

```bash
npm install
```

이 명령어는 `package.json`에 정의된 모든 의존성을 설치합니다.

## 로컬 개발 서버 실행

### 방법 1: 기본 Vite 개발 서버만 실행

가장 간단한 방법입니다. 프론트엔드만 테스트할 때 사용합니다.

```bash
npm run dev
```

**실행 결과:**
- 개발 서버가 시작됩니다
- 기본적으로 `http://localhost:5173`에서 실행됩니다 (포트가 사용 중이면 자동으로 다른 포트 사용)
- 터미널에 접속 URL이 표시됩니다

**특징:**
- 핫 리로드(Hot Module Replacement) 지원
- 코드 변경 시 자동으로 브라우저가 새로고침됩니다
- 빠른 개발 경험 제공

### 방법 2: 프록시 서버와 함께 실행 (입찰공고 검색 기능 테스트용)

입찰공고 검색 기능을 테스트하려면 프록시 서버가 필요합니다.

```bash
npm run start
```

또는

```bash
npm run dev:all
```

**실행 결과:**
- HTTP 서버: `http://localhost:3000`
- 프록시 서버: `http://localhost:3001`
- Vite 개발 서버: `http://localhost:5173` (또는 다른 포트)

**접속 방법:**
- 브라우저에서 `http://localhost:3000/index.html` 접속
- 또는 Vite 개발 서버 URL 접속

### 방법 3: 개별 서버 실행

각 서버를 별도 터미널에서 실행할 수도 있습니다:

**터미널 1 - HTTP 서버:**
```bash
npm run http
```

**터미널 2 - 프록시 서버:**
```bash
npm run server
```

**터미널 3 - Vite 개발 서버:**
```bash
npm run dev
```

## 브라우저에서 확인

### 1. 브라우저 열기

개발 서버가 실행되면 터미널에 표시된 URL을 브라우저에서 열어주세요.

**일반적인 URL:**
- Vite 개발 서버: `http://localhost:5173`
- HTTP 서버: `http://localhost:3000/index.html`

### 2. 개발자 도구 열기

브라우저에서 다음 키를 눌러 개발자 도구를 엽니다:

- **Chrome/Edge**: `F12` 또는 `Ctrl+Shift+I` (Windows/Linux), `Cmd+Option+I` (Mac)
- **Firefox**: `F12` 또는 `Ctrl+Shift+I` (Windows/Linux), `Cmd+Option+I` (Mac)
- **Safari**: `Cmd+Option+I` (Mac, 개발자 메뉴 활성화 필요)

### 3. 확인할 탭

개발자 도구에서 다음 탭을 확인하세요:

- **Console (콘솔)**: JavaScript 오류 및 로그 확인
- **Network (네트워크)**: API 요청 및 응답 확인
- **Elements (요소)**: HTML 구조 확인
- **Sources (소스)**: 소스 코드 디버깅

## 오류 확인 및 디버깅

### 1. 콘솔 오류 확인

브라우저 개발자 도구의 **Console** 탭에서 오류를 확인하세요.

**일반적인 오류 유형:**
- **Syntax Error (문법 오류)**: 코드에 문법 오류가 있을 때
- **TypeError**: 변수나 함수가 예상과 다른 타입일 때
- **ReferenceError**: 정의되지 않은 변수나 함수를 사용할 때
- **Network Error**: API 요청이 실패할 때

### 2. Vite 오류 오버레이

개발 중에 코드에 오류가 있으면 브라우저 화면에 빨간색 오류 오버레이가 표시됩니다.

**오류 오버레이 정보:**
- 오류 메시지
- 오류가 발생한 파일과 줄 번호
- 스택 트레이스

**오류 오버레이 닫기:**
- 오버레이 밖을 클릭
- `Esc` 키 누르기
- 코드를 수정하면 자동으로 사라집니다

### 3. 터미널 오류 확인

Vite 개발 서버를 실행한 터미널에서도 오류 메시지를 확인할 수 있습니다.

**일반적인 터미널 오류:**
- 컴파일 오류
- 포트 충돌 오류
- 모듈을 찾을 수 없음 오류

### 4. 네트워크 오류 확인

**개발자 도구 > Network 탭**에서 다음을 확인하세요:

- API 요청이 성공했는지 (상태 코드 200)
- 요청 URL이 올바른지
- CORS 오류가 없는지
- 응답 데이터가 올바른지

## 빌드 테스트

프로덕션 배포 전에 빌드가 정상적으로 작동하는지 확인하세요.

### 1. 프로덕션 빌드 실행

```bash
npm run build
```

**실행 결과:**
- `dist/` 폴더에 빌드된 파일이 생성됩니다
- 최적화된 JavaScript와 CSS 파일이 생성됩니다
- 빌드 오류가 있으면 터미널에 표시됩니다

### 2. 빌드 결과 확인

빌드가 성공하면 다음을 확인하세요:

- `dist/` 폴더가 생성되었는지
- `dist/index.html` 파일이 있는지
- JavaScript와 CSS 파일이 생성되었는지
- 파일 크기가 합리적인지

### 3. 빌드 오류 해결

빌드 중 오류가 발생하면:

1. 오류 메시지를 자세히 읽으세요
2. 오류가 발생한 파일과 줄 번호를 확인하세요
3. 문법 오류나 import 오류를 수정하세요
4. 다시 빌드를 시도하세요

## 프로덕션 미리보기

빌드된 파일을 로컬에서 미리 볼 수 있습니다.

### 1. 빌드 실행

먼저 프로덕션 빌드를 실행하세요:

```bash
npm run build
```

### 2. 미리보기 서버 실행

```bash
npm run preview
```

**실행 결과:**
- 빌드된 파일을 서빙하는 미리보기 서버가 시작됩니다
- 기본적으로 `http://localhost:4173`에서 실행됩니다
- 터미널에 접속 URL이 표시됩니다

### 3. 브라우저에서 확인

터미널에 표시된 URL을 브라우저에서 열어 프로덕션 빌드가 정상적으로 작동하는지 확인하세요.

## 문제 해결

### 포트가 이미 사용 중일 때

다른 프로세스가 포트를 사용하고 있을 수 있습니다.

**해결 방법:**
1. 해당 포트를 사용하는 프로세스를 종료하세요
2. 또는 `vite.config.js`에서 포트를 변경하세요

### 모듈을 찾을 수 없음 오류

의존성이 제대로 설치되지 않았을 수 있습니다.

**해결 방법:**
```bash
rm -rf node_modules
npm install
```

### 캐시 문제

Vite 캐시 문제일 수 있습니다.

**해결 방법:**
```bash
rm -rf node_modules/.vite
npm run dev
```

### 빌드 오류

빌드 중 오류가 발생하면:

1. 오류 메시지를 확인하세요
2. 문법 오류를 수정하세요
3. import 경로를 확인하세요
4. 다시 빌드를 시도하세요
