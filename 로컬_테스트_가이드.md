## 🎯 빠른 시작

### 1단계: 개발 서버 실행

터미널에서 다음 명령어를 실행하세요:

```bash
cd /Users/donghnc/Documents/bcsa
npm run dev
```

또는 더 간단하게:

```bash
./start-local.sh
```

### 2단계: 브라우저 접속

자동으로 브라우저가 열리거나, 수동으로 다음 주소로 접속하세요:

```
http://127.0.0.1:3000
```

### 3단계: 서버 종료

테스트가 끝나면 터미널에서 `Ctrl + C`를 눌러 서버를 종료하세요.

### 2. 입찰공고 기능 ⭐

**테스트 항목:**
- [ ] 입찰공고 목록 조회
- [ ] 검색 필터 (공고명, 날짜, 공고종류 등)
- [ ] 상세조건 펼치기/접기
- [ ] 검색 결과 표시
- [ ] 초기화 버튼

**테스트 방법:**

#### 2.1 기본 화면 확인
1. 메뉴에서 **「입찰공고」** 클릭
2. 다음 항목이 표시되는지 확인:
   - 입찰공고번호 입력란
   - 공고명 입력란
   - 공고종류 선택 (전체/사전공고/실공고)
   - 공고일자/개찰일자 범위 선택
   - 입찰마감제외 체크박스
   - 「상세조건 펼치기」 링크
   - 초기화, 검색, 상세조건 버튼

#### 2.2 상세조건 펼치기/접기
1. **「상세조건 펼치기」** 클릭
2. 추가 필드 표시 확인:
   - 업무구분, 업무여부
   - 기관명, 참조번호
   - 참가제한지역, 업종
   - 추정가격, 세부품명번호
   - 조달요청번호, 쇼핑몰공고
   - 국내/국제, 계약유형
   - 계약법구분, 계약방법, 낙찰방법
3. **「상세조건 접기」** 또는 하단 버튼으로 다시 접기
4. 필드가 숨겨지는지 확인

#### 2.3 검색 테스트
1. 공고명에 **「대전」** 입력
2. 날짜 범위 설정 (예: 2025-12-30 ~ 2026-01-29)
3. 공고종류 **「실공고」** 선택
4. **「검색」** 버튼 클릭
5. 검색 결과 확인
6. 브라우저 콘솔(F12)에서 API 호출 로그 확인

#### 2.4 초기화 테스트
1. 여러 필드에 값 입력
2. **「초기화」** 버튼 클릭
3. 모든 입력값이 초기화되는지 확인

#### 2.5 API 연동 확인
- 개발자 도구(F12) → Console 탭에서 다음 로그 확인:
  - `DEBUG_G2B_PARAMS`
  - `[TenderSearchFilter]` 관련 로그
- Network 탭에서 API 요청/응답 확인

## 🐛 문제 해결

### 포트가 이미 사용 중인 경우

다른 프로그램이 3000번 포트를 사용 중이면:

```bash
# 다른 포트로 실행 (예: 5173)
npx vite --port 5173
```

그 후 브라우저에서 `http://127.0.0.1:5173` 접속

### Firebase 연결 오류

1. 인터넷 연결 확인
2. Firebase 프로젝트 설정 확인 (`src/firebase.js`)
3. Firebase Console에서 프로젝트 상태 확인

### API 호출 오류

1. `.env.development` 파일 확인
2. API URL이 올바른지 확인: `VITE_API_URL=https://apibid-oytjv32jna-du.a.run.app`
3. 브라우저 콘솔(F12)에서 에러 메시지 확인
4. Network 탭에서 실패한 요청 확인

### 이미지 로딩 오류

1. `public/assets/images/` 폴더에 이미지 파일 존재 확인
2. 이미지 경로가 올바른지 확인
3. 브라우저 캐시 삭제 후 재시도

### 스타일이 적용되지 않는 경우

1. Tailwind CSS 빌드 확인
2. 브라우저 캐시 삭제 (Ctrl + Shift + R)
3. 개발자 도구에서 CSS 로딩 확인

### 주소 검색 (Daum Postcode) 오류

1. `index.html`에 Daum Postcode 스크립트 포함 확인
2. HTTP 서버에서 실행 중인지 확인 (file:// 프로토콜에서는 작동 안 함)
3. 인터넷 연결 확인

## 📊 개발자 도구 활용

### 콘솔 로그 확인

브라우저에서 `F12` 또는 `Cmd+Option+I` (Mac)를 눌러 개발자 도구 열기:

1. **Console 탭**: 에러 메시지, 디버그 로그 확인
2. **Network 탭**: API 요청/응답 확인
3. **Application 탭**: LocalStorage, SessionStorage 확인
4. **Elements 탭**: HTML/CSS 구조 확인

### 유용한 디버그 로그

입찰공고 검색 시 콘솔에 출력되는 로그:
- `DEBUG_G2B_PARAMS`: API 요청 파라미터
- `[TenderSearchFilter]`: 검색 필터 상태
- API 응답 데이터

## 📝 추가 정보

### 환경 변수

개발 환경에서 사용되는 환경 변수 (`.env.development`):
```
VITE_API_URL=https://apibid-oytjv32jna-du.a.run.app
```

### 주요 기술 스택

- **프론트엔드**: React 18, Vite
- **스타일링**: Tailwind CSS
- **라우팅**: React Router v6
- **백엔드**: Firebase (Auth, Firestore, Storage)
- **지도**: Kakao Map API
- **결제**: PortOne (구 아임포트)

### 프로젝트 구조

```
bcsa/
├── src/
│   ├── components/     # 재사용 가능한 컴포넌트
│   ├── pages/          # 페이지 컴포넌트
│   ├── services/       # Firebase 서비스
│   ├── utils/          # 유틸리티 함수
│   ├── constants/      # 상수 정의
│   ├── hooks/          # 커스텀 훅
│   ├── App.jsx         # 메인 앱 컴포넌트
│   └── main.jsx        # 엔트리 포인트
├── public/
│   └── assets/         # 정적 파일 (이미지, CSS, JS)
├── functions/          # Firebase Functions
└── index.html          # HTML 템플릿
```
