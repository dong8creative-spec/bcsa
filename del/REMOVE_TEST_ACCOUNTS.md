# 테스트 계정 삭제 가이드

이 문서는 Firebase Firestore에서 테스트 계정 데이터를 삭제하는 방법을 안내합니다.

## 제거 대상

다음 테스트 계정들을 Firebase에서 삭제해야 합니다:

1. **람람**: 이름(name) 필드가 '람람'인 사용자 계정
2. **admin@busan-ycc.com**: 이메일(email) 필드가 'admin@busan-ycc.com'인 사용자 계정

> **참고**: '햇반' 계정은 운영 계정이므로 삭제하지 마세요.

## Firebase 콘솔에서 계정 삭제 방법

### 1단계: Firebase Console 접속

1. [Firebase Console](https://console.firebase.google.com/)에 접속합니다
2. 해당 프로젝트를 선택합니다

### 2단계: Firestore Database 이동

1. 왼쪽 메뉴에서 **Firestore Database**를 클릭합니다
2. **데이터** 탭이 선택되어 있는지 확인합니다

### 3단계: users 컬렉션 찾기

1. 컬렉션 목록에서 **users** 컬렉션을 찾아 클릭합니다

### 4단계: 테스트 계정 검색 및 삭제

#### 람람 계정 삭제:

1. **users** 컬렉션에서 각 문서를 확인합니다
2. `name` 필드가 **'람람'**인 문서를 찾습니다
3. 해당 문서의 행을 클릭하여 선택합니다
4. 문서 상세 페이지에서 **삭제** 버튼을 클릭합니다
5. 확인 대화상자에서 **삭제**를 클릭합니다

#### admin@busan-ycc.com 계정 삭제:

1. **users** 컬렉션에서 각 문서를 확인합니다
2. `email` 필드가 **'admin@busan-ycc.com'**인 문서를 찾습니다
3. 해당 문서의 행을 클릭하여 선택합니다
4. 문서 상세 페이지에서 **삭제** 버튼을 클릭합니다
5. 확인 대화상자에서 **삭제**를 클릭합니다

### 5단계: Firebase Authentication에서도 삭제 (선택사항)

Firebase Authentication에 등록된 계정도 삭제하려면:

1. 왼쪽 메뉴에서 **Authentication**을 클릭합니다
2. **사용자** 탭을 선택합니다
3. 이메일이 **'admin@busan-ycc.com'**인 사용자를 찾습니다
4. 사용자 옆의 **⋮** 메뉴를 클릭하고 **사용자 삭제**를 선택합니다
5. 확인 대화상자에서 **삭제**를 클릭합니다

> **참고**: '람람' 계정은 Firebase Authentication에 없을 수 있습니다. 이름으로는 검색이 어려우므로 이메일이 있는 경우에만 Authentication에서 삭제하세요.

## 삭제 전 확인사항

⚠️ **주의**: 삭제하기 전에 다음을 확인하세요:

1. **백업**: 중요한 데이터가 있다면 미리 백업하세요
2. **의존성 확인**: 해당 계정이 다른 데이터(게시글, 댓글 등)에 연결되어 있는지 확인하세요
3. **운영 계정 구분**: 운영에 필요한 계정('햇반')은 절대 삭제하지 마세요

## 삭제 후 확인

1. 웹사이트에서 해당 계정으로 로그인 시도 → 실패해야 합니다
2. 관리자 권한 확인 → '람람'과 'admin@busan-ycc.com' 계정은 더 이상 관리자 권한을 가지지 않아야 합니다
3. 회원 목록 확인 → 해당 계정이 목록에서 사라졌는지 확인합니다

## 문제 발생 시

만약 삭제 후 문제가 발생하면:

1. Firebase Console의 **활동 로그**에서 삭제 내역을 확인합니다
2. 필요시 데이터를 복구할 수 있습니다 (30일 이내)
3. Firebase 지원팀에 문의하세요
